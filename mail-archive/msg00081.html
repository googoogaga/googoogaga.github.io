Hi Neel,

I can only provide guidance from having installed it on cygwin but
hopefully it'll help.

I didn't do a separate 'configure' on Boehm. I just ran './configure;
make' on Goo and it picked up the local installation of Boehm for me,
configuring and making it itself.

Do you have GOO_ROOT and GOO_BUILD_ROOT set to the installation
directory? (/tmp/goo in your case).

>From the looks of things its using the dynamic compilation feature
which doesn't work on Win32 so I don't know much about it. Try
evaluating ',ast-eval' to switch to ast evaluation and see if that
works. You'll miss out on the fun of dynamic compilation though.
